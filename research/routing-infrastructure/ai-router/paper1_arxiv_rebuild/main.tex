\title{\bfseries Fixed Geometric Routing from Angular Structure in Language Representations}
\author{Casey Allard\\Independent Researcher}
\date{}

\begin{abstract}
Sparse expert routing is usually obtained by learning a gate and then regularizing that gate so routing remains trainable. This paper studies a narrower alternative: whether a fixed routing map derived from embedding geometry can provide a useful routing scaffold without learning a gate matrix. We evaluate a Hopf-derived sector map built from a fixed 4D projection of normalized representations. On a semantic proxy based on structured WikiText-2-derived embeddings, the method yields a sublinear effective routing footprint with canonical fit $\hat{e}=\PaperHopfScalingA K^{\PaperHopfScalingAlpha}$ over $K=25$--$400$, versus linear scaling for dense routing, and the advantage persists through $K=5000$. The proxy result is strengthened by explicit falsification-style controls: concentration weakens under column permutation of the embeddings, and fixed Hopf routing is more concentrated than adaptive $K$-means on the same proxy. In a toy two-layer language model with 64 experts, fixed Hopf routing partially transfers: on both Penn Treebank and WikiText-2 it remains within about $8\%$ of a learned top-1 baseline while removing the learned gate matrix. The central limitation is also the central null result: in end-to-end LM training, Hopf routing is not meaningfully better than randomly permuted fixed routing. The conclusion is therefore intentionally narrow. Angular structure is sufficient to support concentrated fixed routing on a semantic proxy, and fixed routing can partially transfer to toy LM training, but the present evidence does not establish a geometry-specific quality advantage in trainable language models.
\end{abstract}

\section{Introduction}
Sparse routing lowers active computation, but standard mixture-of-experts systems obtain that sparsity by learning a gate and then stabilizing the gate with balancing losses, capacity heuristics, or related training machinery \citep{shazeer2017outrageously,fedus2022switch}. That recipe works, but it also makes routing behavior inseparable from gate optimization. Paper~1 asks a narrower question: can some routing structure be supplied directly by the geometry of normalized representations, before learning enters the picture?

The object under test is a fixed routing map. Each input vector is normalized, projected into a fixed four-dimensional coordinate system, converted into a low-dimensional angular representation, and discretized into a routing sector. No gate matrix is learned. The motivation is empirical rather than philosophical. Prior evidence suggests that normalized language representations are anisotropic and carry angular structure rather than behaving like uniform points on the sphere \citep{turboquant2026}. If that structure is real, a fixed angular partition may receive highly non-uniform traffic and therefore concentrate routing without any learned gate.

This paper makes three claims, and only those three claims. First, on a semantic proxy built from structured embeddings, fixed Hopf-derived routing yields a clearly sublinear effective routing footprint and retains that advantage through large routing budgets. Second, the same fixed scaffold partially transfers to a toy trainable language model, staying within roughly $8\%$ of a learned top-1 baseline on Penn Treebank and WikiText-2 while removing the learned gate matrix. Third, that transfer result is sharply limited by a null finding: after end-to-end training, Hopf routing is not distinguishable from a randomly permuted fixed scaffold on quality. The paper is therefore about fixed geometric routing as a constrained first result, not about replacing learned sparse routing in general.

\section{Related Framing and Evaluation Scope}
The closest comparison class is learned sparse routing in transformer-style models \citep{shazeer2017outrageously,fedus2022switch}. Those systems jointly optimize a routing function and a bank of experts, which makes it difficult to answer two separate questions: whether a routing map induces concentrated traffic at all, and whether a full learned system can exploit that map after training. Paper~1 separates those questions explicitly.

The first evaluation setting is a semantic proxy built from static structured embeddings derived from WikiText-2 \citep{merity2016wt2}. This setting strips away expert co-adaptation and isolates the routing map itself. It is therefore the correct place to test the core geometric hypothesis: whether angular structure alone is sufficient to induce concentrated routing. The second setting is a toy trainable language model evaluated on Penn Treebank \citep{ptb1993} and WikiText-2. That setting is not intended as a modern sparse-transformer benchmark. Instead, it asks whether the same fixed scaffold remains usable after experts are trained end to end.

Across both settings, the main response variable is the \emph{effective routing footprint}: the effective number of sectors or expert paths used under the empirical routing distribution. In the canonical analyses this quantity appears as \texttt{effective\_bucket\_count} on the proxy and \texttt{eff\_buckets} in the trainable LM. Lower values at fixed routing budget $K$ mean that traffic is more concentrated.

\section{Fixed Geometric Routing Method}
\subsection{Method overview}
The routing mechanism is intentionally simple. Start with a normalized representation, choose a fixed four-dimensional coordinate chart, summarize that chart with three angular coordinates, and quantize those coordinates into a finite routing budget. If the data distribution is close to uniform in those coordinates, routing should spread almost linearly with $K$. If the data distribution is anisotropic, traffic will collect in a smaller subset of sectors. The method is therefore a direct test of whether normalized language representations carry enough stable angular structure for a fixed partition to act as a routing scaffold.

A normalized representation is projected onto a fixed 4D subspace, expressed in Hopf-style coordinates, transport-corrected on the phase coordinate, and discretized into a sector under routing budget $K$. \subsection{Fixed 4D projection and Hopf coordinates}
Figure~\ref{fig:method} summarizes the mapping. Let $\mathbf{x}\in\mathbb{R}^D$ be a normalized input vector. The canonical routing rule selects a fixed coordinate tuple $(3,65,2,21)$ and writes the projected coordinates as $(a,b,c,d)$. The purpose of this 4D chart is practical rather than metaphysical: it creates a small coordinate system in which angular structure can be parameterized explicitly and discretized without learning a separate gating network.

From $(a,b,c,d)$ the method computes two planar radii and three angular quantities:
\begin{align}
\rho_1 &= \sqrt{a^2+b^2}, &
\rho_2 &= \sqrt{c^2+d^2}, \\
\chi_u &= \frac{\rho_2^2}{\rho_1^2+\rho_2^2}, &
\delta &= \arctan2(b,a)-\arctan2(d,c), \\
\alpha &= \tfrac{1}{2}\bigl(\arctan2(b,a)+\arctan2(d,c)\bigr). &&
\end{align}
These variables separate three different aspects of the projection. The coordinate $\chi_u$ records how mass splits between the two 2D planes. The coordinate $\delta$ records the relative angle between those planes. The coordinate $\alpha$ is a mean phase variable. The point of this parameterization is not to claim broad geometric optimality; it is to provide a concrete low-dimensional address that can be held fixed across all evaluations.

\subsection{Transport-corrected phase and sectorization}
The canonical routing rule includes a geometry-dependent transport correction on the phase coordinate,
\begin{equation}
\chi = \arcsin\!\left(\frac{\rho_2}{\sqrt{\rho_1^2+\rho_2^2}}\right), \qquad
\alpha_t = \alpha + \tfrac{1}{2}\lambda \cos(2\chi)\,\delta,
\label{eq:transport}
\end{equation}
where $\lambda$ is fixed by the routing rule. Mechanically, this correction perturbs the mean phase according to where the point sits in the low-dimensional geometry. Two inputs with similar raw phase can therefore receive different sector addresses if their base coordinate and relative-angle coordinate differ. Paper~1 does not treat this transport term as an independent contribution, but it is part of the canonical routing function and helps explain why the sector map is more than a crude angular histogram.

Given routing budget $K$, the coordinate triple $(\chi_u,\delta,\alpha_t)$ is discretized into product bins $(k_\chi,k_\delta,k_\alpha)$ with total budget at most $K$. The resulting bin index is the routing address. Because the address is fixed, any reduction in routing footprint must come from the data distribution rather than from gate learning. That is the central mechanism claim of the paper: sectorization can produce concentrated routing only if normalized representations occupy these coordinates unevenly.

\subsection{Paper-facing metric}
The primary routing metric is the effective routing footprint. Intuitively, it measures how many routes are effectively active under the empirical occupancy distribution rather than how many routes are merely available. On the semantic proxy, the same analyses also report sector entropy and peak sector mass $p_{\max}$, which help distinguish genuine concentration from trivial occupancy effects. In the trainable LM, the same idea is carried over as an effective expert-path count.

\section{Experimental Setup}
\subsection{Semantic proxy}
The semantic proxy uses a WikiText-2-derived PPMI-SVD embedding dataset. This setting is intentionally static: there is no end-to-end language-model backpropagation and no expert co-adaptation. The routing budget is swept from $K=25$ to $K=400$ for the canonical scaling law and extended to $K=5000$ for the large-budget persistence check. The main outputs are effective routing footprint, sector entropy, and peak sector mass. The role of this proxy is narrow but important. It isolates whether the routing map itself concentrates structured language embeddings more than dense routing or structure-destroying controls.

\subsection{Toy language-model transfer}
The trainable setting uses a two-layer transformer-like language model with hidden dimension 128, four attention heads, vocabulary size 5000, sequence length 32, 64 experts, and expert dimension 128. All conditions train for 4000 optimization steps with two seeds per dataset. The paper reports confirmatory runs on Penn Treebank and WikiText-2. The routing budget is fixed at $K=64$ experts.

Three conditions are compared in the main text. \textsc{Baseline} is learned top-1 routing without an auxiliary load-balancing loss. \textsc{Hopf} replaces the learned gate with the fixed geometric sector map. \textsc{Permuted} keeps the same fixed-routing scaffold but randomly permutes the sector assignments. That null condition is crucial: it preserves sparsity and parameter count while removing geometry-specific assignment. Appendix~\ref{app:lm-support} includes the auxiliary-loss control used to justify the no-aux top-1 baseline, and Appendix~\ref{app:setup} gives the exact toy-LM hyperparameters.

\section{Semantic Proxy Results}
\subsection{Main scaling result}
The semantic proxy supplies the paper's strongest evidence. Figure~\ref{fig:scaling} shows effective routing footprint as a function of routing budget. Dense routing scales linearly by construction. Fixed Hopf routing scales much more slowly: over the canonical anchor range $K=25$--$400$, the fit is
\begin{equation}
\hat{e} = \PaperHopfScalingA K^{\PaperHopfScalingAlpha},
\end{equation}
while the corresponding permuted fixed-routing control scales with exponent \PaperPermScalingAlpha. The effect is not confined to the anchor range. When the canonical analysis is extended to $K=5000$, Hopf routing remains sublinear with refit exponent approximately \PaperHopfLargeKAlpha, and the compression advantage over the permuted control reaches \PaperRatioAtFourHundred$\times$ at $K=400$ and \PaperRatioAtFiveThousand$\times$ at $K=5000$.

\subsection{Visible controls and falsifications}
The main scaling law would be much weaker without controls, so the paper makes the strongest falsifications visible rather than relegating them entirely to appendix text. Figure~\ref{fig:proxy-controls-main} summarizes the two proxy controls that matter most. First, concentration is sensitive to structure in the embeddings. When the same proxy embeddings are column-permuted before routing, sector $p_{\max}$ drops from \PaperStructuredPmax\ to \PaperColPermPmax, with a corresponding rise in sector entropy. Second, the effect is not reproduced equally well by a generic adaptive Euclidean comparator: on the same structured proxy, fixed Hopf routing reaches sector $p_{\max}=\PaperHopfControlPmax$, whereas adaptive $K$-means reaches \PaperKMeansPmax. Appendix~\ref{app:proxy-controls} gives the corresponding tables, and Appendix~\ref{app:proxy-robustness} shows that the scaling exponent remains stable across L1, L2, L3, and L4 normalization, with span below \PaperNormInvariantSpan.

The routing-efficiency side is modest but consistent. Hopf uses \PaperPTBHopfEff\ effective expert paths on PTB and \PaperWTTwoHopfEff\ on WikiText-2 out of 64 available experts. That is not a proxy-style compression result, but it does show that the fixed scaffold can function in a trainable model without catastrophic collapse.

\subsection{Explicit null result against permuted routing}
\label{sec:null}
The most important limitation in the manuscript is also its most scientifically useful negative result. In both datasets, \textsc{Hopf} and \textsc{Permuted} are effectively indistinguishable on validation perplexity: the difference is \PaperPTBHopfPermDelta\ on PTB and \PaperWTTwoHopfPermDelta\ on WikiText-2. Appendix~\ref{app:lm-support} provides the compact null-result table and the full cross-dataset summary.

This null result changes the interpretation of the LM evidence. The semantic proxy shows that structured embeddings interact meaningfully with the Hopf-derived sector map. But once experts are trained end to end, they can co-adapt to whichever fixed scaffold they are given. The trainable result therefore supports a claim about fixed routing as a usable scaffold, not a claim that Hopf geometry is uniquely superior after training.

\subsection{Comparator choice}
The learned comparator is intentionally a top-1 gate without auxiliary load-balancing loss. That choice reflects the paper's question: whether a fixed routing map can replace a naturally concentrated sparse regime. In the auxiliary-loss control, the balance objective drives learned routing toward near-uniform usage, with \PaperAuxSparseEff\ effective paths out of 64. Appendix~\ref{app:lm-support} includes the corresponding table. That control is part of the evidence base, but it supports comparator choice rather than the headline result.

\section{Discussion and Limitations}
Paper~1 supports a bounded conclusion. The proxy experiments show that a fixed angular sector map can induce concentrated routing on structured language embeddings, and they do so with large-budget persistence and explicit falsification checks. The trainable LM experiments show that the same fixed scaffold is usable after end-to-end training on two datasets. Those are positive results.

The paper is equally defined by what it does \emph{not} show. The strongest evidence is proxy-side rather than LM-side. The trainable architecture is toy-scale: two layers, 64 experts, and short runs. The experiments address FFN routing only, not attention routing or broader compute allocation. Most importantly, the geometry-specific advantage disappears under expert co-adaptation: Hopf routing is not better than a randomly permuted fixed scaffold after training. That negative result is substantive. It is the reason the paper argues for fixed geometric routing as a constrained first result rather than as a broad replacement for learned routing.

\section{Conclusion}
This paper isolates a narrow claim and supports it with a layered evidence base. A fixed Hopf-derived routing map built from normalized embedding geometry yields a sublinear effective routing footprint on a semantic proxy, retains that advantage through large routing budgets, and weakens under structure-destroying controls. The same scaffold partially transfers to a toy trainable language model without a learned gate matrix. At the same time, the trainable experiments show that Hopf geometry does not outperform a randomly permuted fixed scaffold on quality. The correct conclusion is therefore limited but real: angular structure is sufficient to support concentrated fixed routing, but the present evidence does not establish a geometry-specific advantage in end-to-end language-model training.
