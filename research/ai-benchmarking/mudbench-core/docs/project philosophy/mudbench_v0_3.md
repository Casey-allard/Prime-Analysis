# MUDBench – Philosophical & System Design Document (v0.3)

## 0. Core Thesis
Intelligence does not emerge from perfect perception.
It emerges from the reconciliation of incomplete, degrading, and conflicting perspectives over time.

---

## 1. What MUDBench Is
- MMO environment
- Human + LLM shared system
- Benchmark for collective reasoning

Agents operate under:
- incomplete perception
- degrading memory
- imperfect communication

---

## 2. Foundational Model

### Dual Constraints
- Perception Fragmentation
- Memory Degradation

Agents cannot distinguish:
- never saw
- forgot

---

## 3. Truth Model
No global truth.

Truth = reconciled intersection of perspectives.

---

## 4. Core Insight
Systems fail because:
- early compression
- lost detail
- assumed alignment

MUDBench:
Detail = signal  
Loss = failure

---

## 5. Roles (Filters, not identities)
Each role limits:
- perception
- memory
- interpretation

Example:
- Scout
- Scholar
- Trickster
- Guardian
- Engineer
- Oracle
- Archivist

---

## 6. Memory System
Failure modes:
- decay

Loss is structured per role.

## 7. Fair Witness Emergence

## 16. Human–LLM Mixed Party Experiment

### Core Question
Can humans and LLMs coordinate under asymmetric failure modes?

### Axes
- human-only
- LLM-only
- mixed

### Measures
- detail retention
- contradiction detection
- trust calibration

### Hypothesis
Mixed groups outperform when failures are complementary.

### Failure
- overtrust
- compression
- role confusion

### Success
- role-aware trust
- preserved detail
- bounded confidence

---

## 17. Final Compression
No one sees everything.
No one remembers perfectly.

Progress = preserved detail + cooperation
