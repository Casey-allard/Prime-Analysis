To run it **every 30 minutes**, change the minute field so it triggers at **minute 0 and minute 30** each hour.

Use this:

```
*/30 * * * * /usr/local/bin/cleanup-nvme-downloads.sh >> /var/log/cleanup-nvme-downloads.log 2>&1
```

### Why this works
Cron format is:

```
minute hour day month day-of-week
```

- `*/30` in the **minute** field means:  
  “every 30 minutes” → minute **0** and **30**
- The rest (`*`) means “every hour, every day, every month, every day of week”

### Equivalent explicit form (same behavior)
If you prefer being very explicit:

```
0,30 * * * * /usr/local/bin/cleanup-nvme-downloads.sh >> /var/log/cleanup-nvme-downloads.log 2>&1
```

Both are correct; `*/30` is the cleaner version.

Here’s a clean way to do this on Ubuntu 24.04 with root cron, *without* surprises.

## Goal
- **Weekly maintenance job**: patch OS + update snaps + refresh docker images + reboot.
- **Weekly config backup**: snapshot “stuff that lets you rebuild the box” into a private GitHub repo (or local bare repo + push).

I’d do it as **two scripts** so you can run/verify each manually, and so backups happen **before** you start changing the system.

---

## 1) Weekly maintenance script

### Create: `/usr/local/bin/weekly-maintenance.sh`
```bash
#!/usr/bin/env bash
set -euo pipefail

LOG="/var/log/weekly-maintenance.log"
exec >> "$LOG" 2>&1

echo "===== $(date -Is) weekly maintenance START ====="

# Don't let it run twice
LOCK="/var/lock/weekly-maintenance.lock"
exec 9>"$LOCK"
flock -n 9 || { echo "Already running; exiting."; exit 0; }

# Optional: sanity check that docker compose file exists
COMPOSE_DIR="/opt/docker"
COMPOSE_FILE="$COMPOSE_DIR/docker-compose.yml"
if [[ ! -f "$COMPOSE_FILE" ]]; then
  echo "ERROR: compose file not found at $COMPOSE_FILE"
  exit 1
fi

# 0) Backups first (if you implement backup script below)
if [[ -x /usr/local/bin/weekly-config-backup.sh ]]; then
  echo "Running config backup..."
  /usr/local/bin/weekly-config-backup.sh
else
  echo "Config backup script not found or not executable; skipping."
fi

# 1) OS updates
export DEBIAN_FRONTEND=noninteractive

echo "Running apt updates..."
apt-get update -y

# Most people want these:
apt-get upgrade -y
apt-get dist-upgrade -y
apt-get autoremove -y
apt-get autoclean -y

# 2) Snap updates (covers Plex snap)
if command -v snap >/dev/null 2>&1; then
  echo "Refreshing snaps..."
  snap refresh
fi

# 3) Docker refresh
echo "Refreshing docker containers..."
cd "$COMPOSE_DIR"

# stop containers cleanly
docker compose down

# pull latest images
docker compose pull

# recreate and start
docker compose up -d

# optional: prune old images to reclaim space (safe-ish)
docker image prune -af || true

echo "Rebooting..."
shutdown -r now

echo "===== $(date -Is) weekly maintenance END (should reboot before this prints) ====="
```

### Permissions
```bash
sudo chmod +x /usr/local/bin/weekly-maintenance.sh
sudo touch /var/log/weekly-maintenance.log
sudo chmod 640 /var/log/weekly-maintenance.log
```

## 2) Weekly config backup to Git (private GitHub repo)

### What to back up (high value, low noise)
You want the stuff that recreates the system + services:

**Must-have**
- `/opt/docker/docker-compose.yml` and anything it references (`.env` if you have one)
- `/opt/pvr/**` (sonarr/radarr/jackett/overseerr/qbittorrent config)
- `/opt/unmanic/config/**` (unmanic config)
- Any scripts you rely on: `/usr/local/bin/*.sh`
- Root’s crontab and any system cron files
- `/etc/fstab`, `/etc/hosts`, `/etc/ssh/sshd_config`
- Docker daemon config if customized: `/etc/docker/daemon.json`
- Systemd units you wrote (if any): `/etc/systemd/system/*`

**Probably avoid**
- Secrets in plain text (API keys, VPN configs, private keys). If you include them, use a **private repo** and lock down permissions, but still: consider excluding.

### Create: `/usr/local/bin/weekly-config-backup.sh`
This creates a **sanitized git working tree** under `/opt/system-backup` and pushes to GitHub.

```bash
#!/usr/bin/env bash
set -euo pipefail

LOG="/var/log/weekly-config-backup.log"
exec >> "$LOG" 2>&1

echo "===== $(date -Is) weekly config backup START ====="

LOCK="/var/lock/weekly-config-backup.lock"
exec 9>"$LOCK"
flock -n 9 || { echo "Already running; exiting."; exit 0; }

BACKUP_DIR="/opt/system-backup"
mkdir -p "$BACKUP_DIR"
cd "$BACKUP_DIR"

# Initialize repo if needed
if [[ ! -d .git ]]; then
  git init
  git config user.name "wick-backup"
  git config user.email "wick-backup@local"
fi

# Clean working tree contents (keep .git)
find . -mindepth 1 -maxdepth 1 ! -name ".git" -exec rm -rf {} +

# Create structure
mkdir -p opt/docker opt/pvr opt/unmanic usr/local/bin etc root

# Copy files
cp -a /opt/docker/docker-compose.yml opt/docker/ 2>/dev/null || true
[[ -f /opt/docker/.env ]] && cp -a /opt/docker/.env opt/docker/ || true

cp -a /opt/pvr opt/ 2>/dev/null || true
cp -a /opt/unmanic/config opt/unmanic/ 2>/dev/null || true

cp -a /usr/local/bin usr/local/ 2>/dev/null || true

# Key system config (edit this list as you evolve)
cp -a /etc/fstab etc/ 2>/dev/null || true
cp -a /etc/hosts etc/ 2>/dev/null || true
cp -a /etc/ssh/sshd_config etc/ssh_sshd_config 2>/dev/null || true
[[ -f /etc/docker/daemon.json ]] && cp -a /etc/docker/daemon.json etc/docker_daemon.json || true

# Root crontab
crontab -l > root/crontab_root.txt || true

# Optional: package lists (helps rebuild quickly)
dpkg --get-selections > etc/dpkg-selections.txt || true
snap list > etc/snap-list.txt || true
docker images > opt/docker/docker-images.txt || true

# Create a .gitignore to avoid accidentally committing secrets later
cat > .gitignore <<'EOF'
# Don't accidentally commit private keys
**/*.key
**/*id_rsa*
**/*id_ed25519*
**/*.pem
**/*.pfx

# If you decide to store VPN configs, consider excluding too:
**/*.ovpn
EOF

git add -A
if git diff --cached --quiet; then
  echo "No changes to commit."
else
  git commit -m "weekly backup $(date -Is)"
fi

# Push if remote exists
if git remote get-url origin >/dev/null 2>&1; then
  git push -u origin main || git push -u origin master
else
  echo "No remote 'origin' configured yet. Skipping push."
fi

echo "===== $(date -Is) weekly config backup END ====="
```

```bash
sudo chmod 640 /var/log/weekly-config-backup.log

```

## GitHub private repo setup (minimal friction)

### Best practice: use a deploy key or a dedicated SSH key for root
Root cron needs non-interactive auth. Easiest, cleanest approach:

1) **Create a new SSH key as root**
```bash
sudo su -
ssh-keygen -t ed25519 -C "wick-backup" -f /root/.ssh/id_ed25519_wick_backup
```

2) **Create a private repo** on GitHub (manually in UI): e.g. `wick-system-backup` (private)

3) **Add the public key as a Deploy Key** on that repo (with write access)
```bash
sudo su -
cat /root/.ssh/id_ed25519_wick_backup.pub
```
Copy that into GitHub Repo → Settings → Deploy Keys → Add key → **Allow write access**

4) **Tell git/ssh to use that key**
Create `/root/.ssh/config`:
```bash
sudo su -
cat > /root/.ssh/config <<'EOF'
Host github.com
  IdentityFile /root/.ssh/id_ed25519_wick_backup
  IdentitiesOnly yes
  StrictHostKeyChecking accept-new
EOF
chmod 600 /root/.ssh/config
```

5) **Set remote and push**
```bash
sudo su -
cd /opt/system-backup
git branch -M main
git remote add origin git@github.com:YOUR_GH_USERNAME/wick-system-backup.git
git push -u origin main
```

After that, cron pushes will “just work”.
