# ✅ **STEP 1 — Identify the new disk**
Run:

```bash
lsblk -f
```

Look for the new device, likely something like:

```
sdb    10T   (no filesystem)
```

If it has **no mountpoint** and **no filesystem**, that’s perfect.

---

# ✅ **STEP 2 — Partition the drive (GPT)**
Use `fdisk` or `gdisk`:

```bash
sudo fdisk /dev/sdb
```

Inside fdisk:

- `g` → create new GPT table
- `n` → new partition
- accept defaults
- `w` → write

Your new partition will be something like `/dev/sdb1`.

---

# ✅ **STEP 3 — Format the partition as ext4**
```bash
sudo mkfs.ext4 /dev/sdb1
```

---

# ✅ **STEP 4 — Create a permanent mount point**
Let’s make a mount point Plex and all containers can share:

```bash
sudo mkdir -p /mnt/plexmedia
```

---

# ✅ **STEP 5 — Get the disk UUID**
```bash
sudo blkid /dev/sdb1
```

You'll get something like:

```
UUID="abcd-1234-ef00-9876"
```

Copy it.

---

# ✅ **STEP 6 — Add to /etc/fstab for auto-mounting**
Edit fstab:

```bash
sudo nano /etc/fstab
```

Add this line:

```
UUID=abcd-1234-ef00-9876  /mnt/plexmedia  ext4  defaults  0  2
```

Save and test:

```bash
sudo mount -a
```

Then verify:

```bash
ls -la /mnt/plexmedia
```

---

# ✅ **STEP 7 — Fix permissions for Docker containers**
Most of your containers run as:

- user: `PUID=1000`
- group: `PGID=1000`

So set ownership:

```bash
sudo chown -R 1000:1000 /mnt/plexmedia
```

In the folder where your docker-compose.yml lives:

```bash
docker compose up -d
```

Containers will now point at the new disk.
