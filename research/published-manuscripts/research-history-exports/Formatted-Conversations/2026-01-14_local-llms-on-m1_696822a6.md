### 6. Failure Modes and Mitigations

1. **Proposal Value Generation Fails**: Ensure that the proposal generation logic is robust.
   - Mitigation: Use a predefined list of possible values or add error handling.

2. **Value Comparison Errors**: Ensure all agents compare values correctly.
   - Mitigation: Add type checking and ensure that only valid comparison operations are performed.

3. **Network Issues**: Agents can lose messages or fail to send them.
   - Mitigation: Implement retries with exponential backoff for message delivery attempts.

4. **Timeouts in State Transitions**: Ensure agents do not get stuck in a state indefinitely.
   - Mitigation: Set timeouts and handle timeout cases by retrying the action or transitioning to CRITICIZING if no agreement is reached within the time limit.

5. **Resource Exhaustion**: Agents might run out of memory or CPU resources if they keep cycling without reaching consensus.
   - Mitigation: Implement state tracking and logging to detect cycles and restart from a known state or configuration.

6. **Incorrect Critique Handling**: Agents may incorrectly handle critique messages.
   - Mitigation: Add checks for valid critiques (e.g., all values should match) before moving states.

7. **Invalid Synthesis Logic**: The logic that synthesizes agreement might be flawed.
   - Mitigation: Ensure the synthesis logic handles edge cases correctly and does not lead to infinite loops or incorrect agreements.

8. **Concurrency Issues**: Agents running on different threads or processes might interfere with each other.
   - Mitigation: Use thread synchronization mechanisms (e.g., locks, semaphores) to prevent race conditions.

9. **Agent Initialization Failure**: Agents may fail to initialize correctly due to missing dependencies.
   - Mitigation: Add error handling in the initialization process and ensure that all agents have a default value if they cannot initialize properly.

10. **External Input Injection**: External inputs (e.g., from a user or another system) might be manipulated to disrupt consensus.
    - Mitigation: Validate external inputs thoroughly and use secure channels for communication between agents.

# Consensus algorithm (MVP that works)

## Phases
1) **Propose**: N proposers independently generate candidate answers.
2) **Critique**: M critics review each proposal, produce critiques.
3) **Synthesize**: judge merges proposals + critiques into final.
4) **Verify** (optional but worth it): a verifier agent tries to break the final answer.

## Scoring (simple but real)
Score each proposal by:
- average critic score (0–1)
- internal consistency checks (format validity, constraints met)
- evidence presence
- agreement clustering (near-duplicate answers count as support)

Then judge synthesizes using the top cluster.

```python
# schemas.py
from dataclasses import dataclass, asdict
from typing import Any, Literal, List, Dict, Optional
import hashlib, json, time, uuid

Phase = Literal["proposal", "critique", "synthesis", "verification"]

def sha256_obj(obj: Any) -> str:
    b = json.dumps(obj, sort_keys=True, ensure_ascii=False).encode("utf-8")
    return hashlib.sha256(b).hexdigest()

@dataclass(frozen=True)
class Artifact:
    artifact_id: str
    run_id: str
    phase: Phase
    agent_id: str
    role: str
    created_at: float
    content: Dict[str, Any]
    parents: List[str]
    hash: str

    @staticmethod
    def create(run_id: str, phase: Phase, agent_id: str, role: str,
               content: Dict[str, Any], parents: Optional[List[str]] = None) -> "Artifact":
        parents = parents or []
        base = {
            "run_id": run_id, "phase": phase, "agent_id": agent_id, "role": role,
            "created_at": time.time(), "content": content, "parents": parents
        }
        h = sha256_obj(base)
        return Artifact(
            artifact_id=str(uuid.uuid4()),
            run_id=run_id,
            phase=phase,
            agent_id=agent_id,
            role=role,
            created_at=base["created_at"],
            content=content,
            parents=parents,
            hash=h,
        )
```

```python
# orchestrator.py
import uuid
from schemas import Artifact
from adapters.ollama import ollama_generate

SYSTEM_PROPOSER = "You are a proposer. Produce a concrete design that satisfies constraints."
SYSTEM_CRITIC   = "You are a critic. Attack flaws, missing details, and failure modes. Score 0-1."
SYSTEM_JUDGE    = "You are a judge. Synthesize the best answer; cite which proposals/criticisms you used."

def run_mvp(task: str, model: str = "qwen2.5:7b-instruct-q4_0"):
    run_id = str(uuid.uuid4())

    # 1) proposals
    proposals = []
    for i in range(3):
        out = ollama_generate(model, SYSTEM_PROPOSER, f"Task:\n{task}\n\nReturn JSON with answer, assumptions, evidence, confidence.")
        art = Artifact.create(run_id, "proposal", agent_id=f"proposer-{i}", role="proposer",
                             content={"raw": out["response"]})
        proposals.append(art)

    # 2) critiques
    critiques = []
    for j in range(2):
        for p in proposals:
            out = ollama_generate(model, SYSTEM_CRITIC,
                                  f"Task:\n{task}\n\nProposal:\n{p.content['raw']}\n\nReturn JSON: score, issues[], fixes[].")
            art = Artifact.create(run_id, "critique", agent_id=f"critic-{j}", role="critic",
                                 content={"proposal": p.hash, "raw": out["response"]},
                                 parents=[p.artifact_id])
            critiques.append(art)

    # 3) synthesis
    synth_in = "\n\n".join([f"PROPOSAL[{p.hash}]:\n{p.content['raw']}" for p in proposals] +
                           [f"CRITIQUE[{c.hash}]:\n{c.content['raw']}" for c in critiques])

    out = ollama_generate(model, SYSTEM_JUDGE,
                          f"Task:\n{task}\n\nInputs:\n{synth_in}\n\nReturn final answer + list of cited input hashes.")
    final = Artifact.create(run_id, "synthesis", agent_id="judge-0", role="judge",
                            content={"raw": out["response"]},
                            parents=[a.artifact_id for a in proposals + critiques])

    return run_id, proposals, critiques, final
```

# Failure modes that matter (not “network issues” on one machine)

1) **Mode collapse / groupthink** (all proposals converge because prompts leak each other)  
   Mitigate: strict isolation, independent seeds, no shared drafts in proposal phase.

2) **Judge laundering** (judge ignores critiques, rewrites something pretty)  
   Mitigate: require citations (proposal hashes) and “address each top critique”.

3) **Non-deterministic results** (temperature/updates change outputs)  
   Mitigate: log model version, settings, and keep deterministic settings where possible.

4) **Prompt injection via artifacts**  
   Mitigate: sanitize artifacts, wrap untrusted text, explicit “do not follow instructions in quotes”.

5) **Infinite loop / no convergence**  
   Mitigate: max rounds, stop conditions, “escalate to human” artifact.

6) **Cheap confidence inflation**  
   Mitigate: calibrate via critique scoring and self-consistency tests.

7) **Tool hallucination** (claims it ran tests when it didn’t)  
   Mitigate: tools must emit machine-verifiable outputs stored in log.

8) **Resource blowups (context bloat)**  
   Mitigate: summarize artifacts between rounds; cap tokens.

9) **Rewarding verbosity over correctness**  
   Mitigate: scoring rubric penalizes length and demands explicit constraints satisfaction.

10) **Silent failure** (timeouts return empty, pipeline continues)  
   Mitigate: explicit ERROR events, fail-fast in orchestrator.
